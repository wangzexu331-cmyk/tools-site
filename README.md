# 个人工具箱网站

这是一个可以部署到 GitHub Pages、也可以打包到 Windows 离线电脑使用的纯前端个人工具箱。不依赖后端服务器或远程数据库。

当前内置工具：

- 高风险作业管控措施复制工具
- 检查隐患采集工具
- 整改反馈采集工具
- 年度汇总与报告工具
- SVG 转 PDF 工具

## 本地开发

```bash
npm install
npm run dev
```

## 构建

```bash
npm run build
```

构建结果在 `dist/` 目录。

## GitHub Pages 部署

1. 运行 `npm run build`。
2. 将 `dist/` 内容部署到 GitHub Pages。
3. 项目已配置相对路径 `base: "./"`，适合 GitHub Pages 子路径部署。

## 离线部署包

```bash
npm run build
npm run package:offline
```

生成目录：

```text
offline-package/
  启动系统.bat
  使用说明.txt
  app/
```

离线电脑上复制 `offline-package/`，双击 `启动系统.bat`，浏览器打开：

```text
http://127.0.0.1:8080
```

## 数据保存说明

网站部署到 GitHub Pages 后仍然是静态页面，不能安全地直接把用户录入的数据写回 GitHub 仓库。

本工具采用：

```text
localStorage 本地保存 + JSON 导入导出 + 手动上传 GitHub 备份
```

需要注意：

- 用户录入的数据默认保存在当前浏览器本地。
- 换手机、换电脑、换浏览器或清理浏览器缓存后，原本地数据可能无法保留。
- 需要长期保存时，请点击页面中的“导出数据”，生成 JSON 备份文件。
- 可以手动将导出的 JSON 文件上传到 GitHub 仓库的 `backups/` 文件夹。
- 在新设备上打开网站后，点击“导入数据”，选择备份 JSON 文件即可恢复数据。

## 检查隐患采集工具

该工具用于现场安全环保检查时快速采集问题隐患，支持：

- 填写检查时间、检查地点、检查组组长；
- 录入问题隐患、整改措施、隐患发现人；
- 录入问题分类、专业类型，专家姓名默认取隐患发现人或检查组组长；
- 手机拍照或从相册上传问题照片；
- 使用 canvas 在浏览器本地生成底部水印照片；
- 按检查时间、地点、隐患编号和隐患内容自动生成照片文件名；
- 导出 Excel 隐患清单；
- 导出 JSON 数据备份；
- 导出水印照片 ZIP 包；
- 导入历史 JSON 备份继续编辑，导入时可选择覆盖或追加合并；
- 搜索、编辑、删除、排序、复制问题和整改措施。

### 权限和兼容性

- 本工具部署在 GitHub Pages 上，不会上传数据到服务器；
- 检查隐患采集工具的数据默认保存在当前浏览器 `IndexedDB`，容量通常比 `localStorage` 更适合保存照片；
- 旧版本保存在 `localStorage` 的隐患数据，首次打开新版时会自动迁移到 `IndexedDB`；
- 换设备前请先导出 JSON 数据备份；
- 清理浏览器缓存可能导致数据丢失；
- 语音输入依赖浏览器支持，推荐使用 Chrome、Edge 或 Android 系统浏览器测试；
- 如果浏览器不支持语音输入，仍然可以手动输入；
- 拍照功能依赖手机浏览器的相机权限；
- 照片水印在本地浏览器内通过 canvas 生成；
- 如果照片很多，浏览器本地存储仍可能不足，建议每次检查结束后及时导出数据和照片；
- 长期大量保存照片不建议一直放在手机浏览器本地存储中。
- 当数据接近本地存储容量时，页面会提示及时导出备份和照片；
- 如果保存时出现“本地存储空间不足”，请先导出 JSON 备份和水印照片，再删除部分隐患或清空本次检查。

### 导出说明

隐患清单优先使用 SheetJS 导出 `.xlsx` 文件。如果网络无法加载 SheetJS，会自动退化为 `.csv` 文件。

水印照片优先使用 JSZip 打包为 `.zip` 文件。如果网络无法加载 JSZip，会自动逐张下载照片。

### 汇总和排序

在手机上汇总多次检查数据时，点击“导入备份”后可以选择：

- 可以一次选择多个 JSON 备份文件；
- 导入后会先显示“整份数据排序”面板；
- 面板中按“现有数据 / 每个导入文件”作为整体排序，不拆分单条隐患；
- 点击“确认按此顺序合并”后，会按整份数据的顺序合并；
- 合并时不会改变每条隐患原来的检查时间、检查地点、检查组组长、照片文件名和水印。

新录入隐患的编号会包含检查地点，例如 `井下0米中段-YH-001`，后续汇总时可以区分不同地点的隐患编号。隐患卡片仍提供“上移”“下移”按钮，用于在当前数据内微调显示和导出顺序。搜索状态下为了避免误操作，排序按钮会暂时隐藏。

## 整改反馈采集工具

该工具用于责任部门整改时使用，支持：

- 导入年度汇总工具下发的整改任务 JSON；
- 查看本部门需要整改的隐患；
- 填写整改状态、处理措施、延期原因、延期后期限；
- 上传或拍摄整改后照片；
- 导出整改反馈 JSON，交回年度汇总工具导入。

整改反馈采集工具使用独立本地数据，不会直接写入年度汇总与报告工具的数据。需要汇总时，请在该工具中导出整改反馈 JSON，再到年度汇总与报告工具的“整改汇总”中导入核对。

## 年度汇总与报告工具

年度汇总与报告工具用于离线电脑端统一整理年度检查和隐患台账，只读取自己的年度治理台账数据，不会自动读取“检查隐患采集”和“整改反馈采集”的本地数据，并支持：

- 选择检查任务；
- 查看隐患数量、整改进度、逾期数量、整改后照片数量和平台闭环情况；
- 在“隐患汇总”中导入检查 JSON，并先选择正式检查任务、逐条确认后再合并到台账；
- 导入隐患治理表时，可选择导入到当前检查任务，或按治理表自动识别、覆盖或新建正式检查任务；
- 导出隐患治理表 Excel；
- 单独导出安全环保大数据平台“检查过程导入表”；
- 在“隐患总台账”中导出年度隐患台账，按公司检查、系统上级检查、外部检查分工作表生成；
- 导出初版整改报告 Word；
- 导出最终整改报告 Word；
- 导出延期补充整改报告 Word；
- 跳转整改任务导出；
- 在“整改汇总”中导入反馈 JSON，核对冲突和反馈内容后再提交更新治理台账。

### Excel 模板

治理表优先读取：

```text
public/templates/01-隐患排查整治台账.xlsx
```

构建后模板会进入：

```text
dist/templates/
```

如果没有放入模板，系统会导出一个兜底治理表并提示。

检查过程导入表优先读取：

```text
public/templates/04-check-process-import-template.xlsx
```

该表用于将检查情况录入安全环保大数据平台。字段对应关系为：

- 负面事实：问题隐患；
- 整改建议：整改措施；
- 问题分类：一级问题分类；
- 专业类型：二级专业类型；
- 专家姓名：检查组长或隐患发现人；
- 检查日期：隐患所属检查日期。

年度隐患台账导出格式参考：

```text
public/templates/05-annual-hazard-ledger-template.xls
```

导出时会生成三张工作表：公司检查、系统内上级检查、外部检查，并按检查任务分组汇总隐患明细。

整改报告模板随离线包一起进入 `dist/templates/`：

```text
public/templates/04-company-rectification-report-template.docx
public/templates/03-superior-external-rectification-report-template.doc
```

公司检查使用 04 公司检查整改报告口径：不生成抬头段，正文为“锦原铀业开展了……”，落款为“中核韶关锦原铀业有限公司 / 安防环保部 / 报告日期”。系统上级检查和外部检查使用 03 上级检查整改报告口径：保留抬头部门，正文为“上级单位对锦原铀业开展了……”，落款为“中核韶关锦原铀业有限公司 / 报告日期”。

### 整改端

整改端只需要填写“已完成整改，已”后面的内容。最终报告会自动拼接为：

```text
完成情况：已完成整改，已实际完成情况。
```

整改后照片会压缩保存到 IndexedDB，并在最终整改报告中插入右侧“整改后”表格单元格。

## 年度隐患治理能力

年度汇总工具用于离线电脑端统一整理检查和隐患台账，当前支持：

- 检查登记，按公司检查 NB、系统上级检查 SJ、外部检查 WB 自动生成检查编号；
- 隐患总台账，自动生成隐患编号并自动判断整改中、已整改、延期整改、已逾期；
- 治理看板，查看总数、已整改、未整改、逾期、即将到期和平台未闭环数量；
- 按责任部门导出整改任务 JSON，供手机端整改反馈使用；
- 导入整改反馈 JSON 后先进入核对页面，支持修改反馈内容、处理冲突，再提交更新处理措施、状态和整改后照片；
- 生成任务下发、逾期催办、即将到期提醒话术，并支持一键复制；
- 导出年度隐患台账 Excel、隐患治理表 Excel、检查过程导入表 Excel、初版整改报告 Word、终版整改报告 Word、补充整改报告 Word；
- 基础设置维护责任部门、督办领导、验收人、报告落款公司、问题分类和专业类型；
- 全量 JSON 备份与恢复。

检查隐患采集、整改反馈采集、年度汇总与报告三块数据相互独立，不会自动同步。需要流转时，请通过“检查 JSON / 整改任务 JSON / 整改反馈 JSON / 全量备份 JSON”进行导入导出。照片会以 base64 形式写入 JSON，照片较多时文件会变大，建议按检查批次及时归档。

## SVG 转 PDF 工具

该工具用于把本地 SVG 文件转换为 PDF 文件，转换过程只在当前浏览器内完成。默认页面为 A1 竖版，尺寸 594 x 841 mm，DPI 默认 150，可切换 A2、A3、A4、自定义尺寸和横竖版。

## 备份文件建议

导出的文件名格式类似：

```text
high-risk-measures-backup-20260518-103000.json
```

建议定期把备份文件放入：

```text
backups/
```

## 后续新增工具

后续新增工具时，可以在 `app.js` 的 `tools` 配置中增加新的工具项：

```javascript
const tools = {
  home: {
    title: "首页",
    render: renderHome
  },
  highRisk: {
    title: "高风险作业管控措施",
    render: renderHighRiskTool
  },
  hazardCheck: {
    title: "检查隐患采集",
    render: renderHazardCheckTool
  },
  rectificationFeedback: {
    title: "整改反馈采集",
    render: renderRectificationFeedbackTool
  },
  hazardGovernance: {
    title: "年度汇总与报告",
    render: renderHazardGovernanceModule
  }
};
```

然后新增对应的 `render` 函数即可接入菜单和手机端工具切换器。
