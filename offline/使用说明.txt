隐患整改报告生成系统离线版使用说明

1. 本系统支持 GitHub Pages 在线版和 Windows 离线版。
2. 离线版不需要互联网，不使用远程服务器或在线数据库。
3. 离线版已内置运行环境，不需要安装 Node.js、Python 或其他运行环境。
4. 优先双击“Start-System.cmd”。也可以双击“启动系统.bat”，它会打开一个不会自动关闭的命令窗口。
5. 浏览器会打开 http://127.0.0.1:8080。如果 8080 被占用，系统会自动改用 8081、8082 等端口，具体地址以启动窗口显示为准。
6. 请不要关闭启动窗口，关闭窗口后系统会停止。
7. 如果启动失败，窗口不会直接闪退，请查看同目录下生成的“startup-log.txt”。
8. 数据保存在本机浏览器 IndexedDB 中。
9. 更换电脑或浏览器前，请先导出检查数据 JSON 和整改反馈 JSON。
10. 在线版和离线版数据不会自动同步，需要通过 JSON 导入导出进行数据流转。
11. 不要删除 app/templates 中的 Excel 模板文件。
12. 不要删除 app、runtime、server.cjs、server.ps1，启动系统.bat 需要调用它们启动本地服务。
13. 生成 Word 或 Excel 前，请确认隐患、整改措施、照片和整改反馈数据完整。
14. 如果窗口一闪而过，通常是文件没有完整解压，或杀毒软件拦截了 runtime\node.exe。请先解压整个文件夹后再双击，不要在压缩包里面直接运行。
15. “Open-Direct-HTML.cmd”和“打开系统.html”是应急入口，优先使用“Start-System.cmd”，因为生成治理表和报告时需要读取模板文件。
